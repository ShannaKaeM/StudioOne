4. Mathematical Relationships (Not Tokens)

Variables Can Have Mathematical Relationships

/* Base values - just numbers */
--base: 1rem;
--ratio: 1.25;

/* Calculated values - still just numbers */
--size-1: calc(var(--base) / var(--ratio));
--size-2: var(--base);
--size-3: calc(var(--base) * var(--ratio));
--size-4: calc(var(--base) * var(--ratio) * var(--ratio));
--size-5: calc(var(--base) * var(--ratio) * var(--ratio) * var(--ratio));

/* These aren't "font sizes" or "spacing" - they're just calculated numbers */

5. How Presets Work

The Magic

A preset doesn't say "make this a button" or "style this as a header".
A preset just says "set these variables to these values".
The zone transforms based on those values. Period.

/* User labels zones for their reference - NOT for the system */
<div class="zone a" data-label="My Navigation">

/* System doesn't care about the label - only presets matter */
.zone.a {
  /* Has all 103 variables available */
  /* Preset will override specific ones */
}

/* Apply preset */
<div class="zone a preset-navbar" data-label="My Navigation">

/* Preset just sets variables */
.preset-navbar {
  --height: 60px;
  --background: var(--color-1);
  --padding: var(--space-3);
  /* That's it - no semantic meaning */
}
